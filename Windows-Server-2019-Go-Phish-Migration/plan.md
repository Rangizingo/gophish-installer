# GoPhish Deployment — Windows Server 2019 (Datacenter)

## Context

GoPhish currently runs in Docker on a local Windows 10 PC. This setup requires Cloudflare quick tunnels for external access, but port 7844 is blocked at the office, making tunnels unreliable. The datacenter's Windows Server 2019 has port 7844 open and a public IP (behind NAT). Deploying there with a Cloudflare **named tunnel** gives a permanent, stable URL with no inbound port forwarding needed.

## Target Server

| Spec | Value |
|------|-------|
| OS | Windows Server 2019 Standard |
| RAM | 8 GB |
| CPU | 2 logical processors |
| Disk | 59.4 GB (20.1 GB free) |
| Docker | Not installed (not needed) |
| Ports 80/443 | Not in use |
| Port 7844 outbound | Confirmed open |
| Public IP | Yes (behind NAT) |

## Architecture

```
Internet → portal.expertimportersllc.com (CNAME → Cloudflare Tunnel)
         → Cloudflare Edge (TLS on tunnel transport)
         → Cloudflare Tunnel (outbound from server, port 7844)
         → localhost:80 (GoPhish phish server)

Admin UI → https://<server-ip>:3333 (self-signed, restricted to your IPs via firewall)
         → Also accessible via RDP on https://127.0.0.1:3333
```

## Phase 1: Install GoPhish (Native Binary, No Docker)

GoPhish's Docker image is Linux-based. Instead of Docker complexity on Windows Server, run the native Windows binary directly.

### 1.1 Download GoPhish

```powershell
# Create directory
New-Item -ItemType Directory -Path "C:\scripts\gophish" -Force
New-Item -ItemType Directory -Path "C:\scripts\gophish\logs" -Force
New-Item -ItemType Directory -Path "C:\scripts\gophish\backup" -Force

# Download latest Windows 64-bit release
$release = Invoke-RestMethod "https://api.github.com/repos/gophish/gophish/releases/latest"
$asset = $release.assets | Where-Object { $_.name -match "windows-64bit" }
Invoke-WebRequest -Uri $asset.browser_download_url -OutFile "C:\scripts\gophish\gophish.zip"

# Extract
Expand-Archive -Path "C:\scripts\gophish\gophish.zip" -DestinationPath "C:\scripts\gophish" -Force
Remove-Item "C:\scripts\gophish\gophish.zip"
```

### 1.2 Configure GoPhish (`C:\scripts\gophish\config.json`)

```json
{
    "admin_server": {
        "listen_url": "0.0.0.0:3333",
        "use_tls": true,
        "cert_path": "gophish_admin.crt",
        "key_path": "gophish_admin.key"
    },
    "phish_server": {
        "listen_url": "0.0.0.0:80",
        "use_tls": false
    },
    "db_name": "sqlite3",
    "db_path": "gophish.db",
    "contact_address": ""
}
```

Key decisions:
- Admin binds to `0.0.0.0:3333` — accessible remotely, but firewall-restricted to your IPs only (Phase 3)
- Phish server binds to `0.0.0.0:80` — cloudflared forwards tunnel traffic here
- Self-signed cert auto-generated by GoPhish on first run
- Day-to-day management: RDP for setup, remote scripts from office/home for campaigns

### 1.3 Install NSSM and Register GoPhish as a Windows Service

```powershell
# Download NSSM (Non-Sucking Service Manager)
Invoke-WebRequest -Uri "https://nssm.cc/release/nssm-2.24.zip" -OutFile "C:\scripts\gophish\nssm.zip"
Expand-Archive -Path "C:\scripts\gophish\nssm.zip" -DestinationPath "C:\scripts\gophish" -Force
Copy-Item "C:\scripts\gophish\nssm-2.24\win64\nssm.exe" "C:\scripts\gophish\nssm.exe"
Remove-Item "C:\scripts\gophish\nssm.zip"
Remove-Item "C:\scripts\gophish\nssm-2.24" -Recurse

# Register as service
C:\scripts\gophish\nssm.exe install gophish "C:\scripts\gophish\gophish.exe"
C:\scripts\gophish\nssm.exe set gophish AppDirectory "C:\scripts\gophish"
C:\scripts\gophish\nssm.exe set gophish AppStdout "C:\scripts\gophish\logs\gophish-stdout.log"
C:\scripts\gophish\nssm.exe set gophish AppStderr "C:\scripts\gophish\logs\gophish-stderr.log"
C:\scripts\gophish\nssm.exe set gophish AppRotateFiles 1
C:\scripts\gophish\nssm.exe set gophish AppRotateBytes 10485760
C:\scripts\gophish\nssm.exe set gophish Start SERVICE_AUTO_START

# Start service
Start-Service gophish
```

### 1.4 First Login

```powershell
# Get temporary admin password from stdout log
Select-String -Path "C:\scripts\gophish\logs\gophish-stdout.log" -Pattern "Please login with"
```

- RDP into the server, open browser to `https://127.0.0.1:3333`
- Accept self-signed cert warning
- Login with `admin` / `<temp password from log>`
- Change password immediately
- Copy the new API key from Settings

### 2.2 Copy Backup Files to Server

Copy from local PC to server via RDP clipboard or file share:

```
Local:  C:\Users\pblanco\Documents\AI\gophish-installer\backup\
Server: C:\scripts\gophish\backup\
```

Files to copy: `templates.json`, `pages.json`, `smtp.json`, `groups.json`

### 2.3 Run Restore Script

Create `C:\scripts\gophish\restore.ps1` on the server with this content.
Replace `<YOUR_NEW_API_KEY>` with the API key from step 1.4:

```powershell
Add-Type @"
using System.Net;
using System.Security.Cryptography.X509Certificates;
public class TrustRestore : ICertificatePolicy {
    public bool CheckValidationResult(ServicePoint s, X509Certificate c, WebRequest r, int p) { return true; }
}
"@
[System.Net.ServicePointManager]::CertificatePolicy = New-Object TrustRestore
[Net.ServicePointManager]::SecurityProtocol = [Net.SecurityProtocolType]::Tls12

$key = "<YOUR_NEW_API_KEY>"
$base = "https://127.0.0.1:3333/api"
$backupDir = "C:\scripts\gophish\backup"

# Restore templates
Write-Host "Restoring templates..." -NoNewline
$templates = Get-Content "$backupDir\templates.json" -Raw | ConvertFrom-Json
foreach ($t in $templates) {
    $body = @{
        name = $t.name
        subject = $t.subject
        text = $t.text
        html = $t.html
        envelope_sender = $t.envelope_sender
        attachments = @()
    } | ConvertTo-Json -Depth 5
    try {
        Invoke-RestMethod -Uri "$base/templates/?api_key=$key" -Method Post -Body $body -ContentType "application/json" | Out-Null
        Write-Host " OK ($($t.name))"
    } catch { Write-Host " FAILED: $($_.Exception.Message)" }
}

# Restore landing pages
Write-Host "Restoring landing pages..." -NoNewline
$pages = Get-Content "$backupDir\pages.json" -Raw | ConvertFrom-Json
foreach ($p in $pages) {
    $body = @{
        name = $p.name
        html = $p.html
        capture_credentials = $p.capture_credentials
        capture_passwords = $p.capture_passwords
        redirect_url = $p.redirect_url
    } | ConvertTo-Json -Depth 5
    try {
        Invoke-RestMethod -Uri "$base/pages/?api_key=$key" -Method Post -Body $body -ContentType "application/json" | Out-Null
        Write-Host " OK ($($p.name))"
    } catch { Write-Host " FAILED: $($_.Exception.Message)" }
}

# Restore groups
Write-Host "Restoring groups..." -NoNewline
$groups = Get-Content "$backupDir\groups.json" -Raw | ConvertFrom-Json
foreach ($g in $groups) {
    $targets = @()
    foreach ($t in $g.targets) {
        $targets += @{
            email = $t.email
            first_name = $t.first_name
            last_name = $t.last_name
            position = $t.position
        }
    }
    $body = @{
        name = $g.name
        targets = $targets
    } | ConvertTo-Json -Depth 5
    try {
        Invoke-RestMethod -Uri "$base/groups/?api_key=$key" -Method Post -Body $body -ContentType "application/json" | Out-Null
        Write-Host " OK ($($g.name))"
    } catch { Write-Host " FAILED: $($_.Exception.Message)" }
}

# Restore SMTP profiles (without password - enter manually in UI)
Write-Host "Restoring SMTP profiles..." -NoNewline
$smtps = Get-Content "$backupDir\smtp.json" -Raw | ConvertFrom-Json
foreach ($s in $smtps) {
    $body = @{
        name = $s.name
        interface_type = $s.interface_type
        host = $s.host
        from_address = $s.from_address
        username = $s.username
        password = ""
        ignore_cert_errors = $s.ignore_cert_errors
        headers = @()
    } | ConvertTo-Json -Depth 5
    try {
        Invoke-RestMethod -Uri "$base/smtp/?api_key=$key" -Method Post -Body $body -ContentType "application/json" | Out-Null
        Write-Host " OK ($($s.name)) [PASSWORD BLANK - update manually in UI]"
    } catch { Write-Host " FAILED: $($_.Exception.Message)" }
}

Write-Host "`nRestore complete. Update SMTP password in GoPhish UI → Sending Profiles."
```

Run it:
```powershell
powershell -ExecutionPolicy Bypass -File C:\scripts\gophish\restore.ps1
```

## Phase 3: Windows Firewall

### 3.1 Allow GoPhish Phish Server (Port 80)

```powershell
New-NetFirewallRule -DisplayName "GoPhish Phish Server (HTTP)" `
    -Direction Inbound -LocalPort 80 -Protocol TCP -Action Allow
```

### 3.2 Restrict Admin UI to Your IPs Only

Admin binds to `0.0.0.0:3333` for remote management, but only your specific IPs are allowed:

```powershell
# Block 3333 from everyone by default
New-NetFirewallRule -DisplayName "GoPhish Admin - Block All" `
    -Direction Inbound -LocalPort 3333 -Protocol TCP -Action Block

# Allow 3333 from your office and home IPs only
New-NetFirewallRule -DisplayName "GoPhish Admin - Management IPs" `
    -Direction Inbound -LocalPort 3333 -Protocol TCP -Action Allow `
    -RemoteAddress "174.105.36.233","70.61.175.62"
```

This lets you run `email-admin-gui.ps1` and campaign scripts from your desk pointing at `https://<server-ip>:3333/api`.

### 3.3 No Inbound Port Forwarding Needed

Cloudflare Tunnel connects **outbound** on port 7844. No NAT/port forwarding rules needed at the datacenter. Traffic flows:

```
Visitor → Cloudflare Edge → (tunnel, outbound from server) → localhost:80
```

## Phase 4: Cloudflare Named Tunnel

### 4.1 Install cloudflared

```powershell
$url = "https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-windows-amd64.msi"
Invoke-WebRequest -Uri $url -OutFile "C:\cloudflared-setup.msi"
msiexec /i C:\cloudflared-setup.msi /quiet /norestart
& "C:\Program Files (x86)\cloudflared\cloudflared.exe" --version
```

### 4.2 Authenticate with Cloudflare

**Important:** `cloudflared tunnel login` requires selecting a Cloudflare zone (domain). Since `expertimportersllc.com` is NOT on Cloudflare (DNS is at Epik), you have two choices:

1. **Add `expertimportersllc.com` to your Cloudflare account** (free plan — doesn't require changing nameservers for tunnel auth, just needs to exist as a zone). Then select it during login.
2. **Use any existing Cloudflare zone** you have (e.g. `blancoitsolutions.com`) — the tunnel itself works with any domain via CNAME. The auth cert just needs to be tied to *a* Cloudflare account.

**Option A — Server has a browser (via RDP):**
```powershell
& "C:\Program Files (x86)\cloudflared\cloudflared.exe" tunnel login
# Browser opens → log into Cloudflare → select any zone you own
# Saves cert to: C:\Users\<username>\.cloudflared\cert.pem
```

**Option B — Headless (authenticate locally, copy cert):**
- Run `cloudflared tunnel login` on local PC
- Copy `%USERPROFILE%\.cloudflared\cert.pem` to server's `%USERPROFILE%\.cloudflared\`

### 4.3 Create Named Tunnel

```powershell
& "C:\Program Files (x86)\cloudflared\cloudflared.exe" tunnel create gophish-portal
# Output: Created tunnel gophish-portal with id <TUNNEL_UUID>
# Also creates: %USERPROFILE%\.cloudflared\<TUNNEL_UUID>.json
```

Save the `<TUNNEL_UUID>` — you need it for the next two steps.

### 4.4 Create Tunnel Config

Copy cert.pem and `<TUNNEL_UUID>.json` to the SYSTEM profile (where the service runs):

```powershell
$tunnelId = "<TUNNEL_UUID>"  # replace with actual UUID from step 4.3

# Create SYSTEM profile directory
$sysDir = "C:\Windows\System32\config\systemprofile\.cloudflared"
New-Item -ItemType Directory -Path $sysDir -Force

# Copy credentials
Copy-Item "$env:USERPROFILE\.cloudflared\cert.pem" "$sysDir\cert.pem"
Copy-Item "$env:USERPROFILE\.cloudflared\$tunnelId.json" "$sysDir\$tunnelId.json"

# Write config
@"
tunnel: $tunnelId
credentials-file: $sysDir\$tunnelId.json

ingress:
  - hostname: portal.expertimportersllc.com
    service: http://localhost:80
  - service: http_status:404
"@ | Set-Content "$sysDir\config.yml"
```

The ingress routes **only** `portal.expertimportersllc.com` to GoPhish port 80. Admin port 3333 is deliberately excluded — never exposed through the tunnel.

### 4.5 Create DNS Route (Epik DNS Console)

Since `expertimportersllc.com` DNS is managed at Epik (not Cloudflare), you add the CNAME manually:

1. Log into **Epik** → Domain Manager → `expertimportersllc.com` → DNS Zone Editor
2. Go to the **CNAME** tab
3. Add a new record:
   - **Host:** `portal`
   - **Points to:** `<TUNNEL_UUID>.cfargotunnel.com`
   - **TTL:** 300 (or default)
4. Save

This routes `portal.expertimportersllc.com` through Cloudflare's tunnel infrastructure without moving the domain to Cloudflare.

**Verify DNS propagation:**
```powershell
nslookup portal.expertimportersllc.com
# Should return CNAME → <TUNNEL_UUID>.cfargotunnel.com
```

### 4.6 Install as Windows Service

```powershell
& "C:\Program Files (x86)\cloudflared\cloudflared.exe" service install
Start-Service cloudflared
Get-Service cloudflared
```

Service properties:
- Startup: Automatic
- Account: LocalSystem
- Auto-reconnects on network interruptions
- Survives reboots

### 4.7 Set Service Dependency

```powershell
# cloudflared waits for GoPhish to start first
sc.exe config cloudflared depend= gophish
```

## Phase 5: SSL/TLS

Since the domain is on Epik (not Cloudflare), Cloudflare's edge TLS termination doesn't apply automatically. Two options:

**Option A — HTTP only (recommended for phishing sim):**
- Campaign URL: `http://portal.expertimportersllc.com`
- No SSL cert needed on the server
- GoPhish phish server already listens on HTTP port 80
- Cloudflare Tunnel encrypts the transport between Cloudflare's edge and your server regardless
- The CNAME to `cfargotunnel.com` means traffic flows through Cloudflare's network with TLS on the tunnel layer

**Option B — HTTPS with Let's Encrypt (more realistic phishing URL):**
- Install Caddy or win-acme on the server as a reverse proxy
- Caddy auto-provisions Let's Encrypt certs for `portal.expertimportersllc.com`
- Caddy listens on 443, proxies to GoPhish on localhost:80
- Adds complexity but gives `https://` in the URL

**Recommendation: Start with Option A.** You can always add HTTPS later if needed. The actual tunnel transport is encrypted either way.

## What This Eliminates

- Docker Desktop dependency
- Cloudflare quick tunnels (random URLs, dying processes)
- Port 7844 office firewall issues
- LAN IP workarounds
- Manual tunnel restarts

## What's Permanent

- `portal.expertimportersllc.com` — stable URL, never changes
- GoPhish runs as a Windows service — auto-start, auto-restart
- Cloudflare Tunnel as a Windows service — auto-reconnect, survives reboots
- No inbound ports needed at datacenter
